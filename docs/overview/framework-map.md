# Framework Map

Use this page when you need the shortest path through the playbook.

## Phase map

| Chapter | Questions answered | Output |
| --- | --- | --- |
| Company research brief | Where does the company make money, lose time, and carry risk? | Business context and operating constraints |
| Workflow discovery | Which workflows repeat enough to matter? | Task inventory and workflow map |
| Task scoring | Which tasks should stay human-led and which should move toward automation? | Human Value Score, Automation Fit Score, and posture decision |
| Automation prioritization | Which candidate should become the next pilot? | Prioritized automation backlog |
| Pilot rollout and governance | How do we deploy without losing trust or control? | Pilot plan, approvals, metrics, and governance loop |

## Decision gates

- Do not score tasks until the workflow and owner are clear.
- Do not prioritize a candidate until human approval boundaries are explicit.
- Do not launch a pilot without security notes, failure modes, and rollback
  criteria.
- Do not treat a successful draft or summary as proof that the workflow is
  production-ready.

## Common anti-patterns

- starting with a model or vendor instead of a workflow
- bundling too many adjacent tasks into one "AI use case"
- ignoring reversibility and later discovering the workflow cannot be audited
- publishing examples that sound plausible but are not grounded in real work
