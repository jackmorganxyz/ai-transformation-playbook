# Key Concepts

Before diving in, here are the terms you will see throughout this playbook — defined in plain language.

## Augmentation

Using AI to make an employee faster and more effective at their existing job — not to replace them. The human stays in control of decisions; the AI handles repetitive execution. A $100k employee with a $10k AI tool producing 2–3x the output is augmentation in action.

## Workflow

A repeatable sequence of tasks that moves a piece of work from start to finish. "Process a customer refund" is a workflow. "Publish a blog post" is a workflow. This playbook asks you to break every role into its workflows so you can see exactly where AI fits.

## Task

A single step inside a workflow. Workflows are made of tasks. "Draft the refund email" is a task within the refund workflow. Some tasks require judgment (deciding whether to approve the refund). Others are pure execution (copying the order number into a template). AI handles the execution tasks.

## Agent

A configured AI system that performs one or more tasks within a workflow — automatically or semi-automatically. An agent is not a chatbot you talk to. It is a defined piece of automation with a clear trigger, a set of inputs, a set of actions, and a defined output. Think of it as a digital teammate that handles a specific job.

## Scoring

The process of evaluating each task for how well AI can handle it. This playbook uses a simple rubric that considers factors like data availability, error tolerance, and judgment required. High-scoring tasks are candidates for AI. Low-scoring tasks stay with humans.

## Shadow Mode

The first deployment stage for any AI agent. The agent runs alongside the human and produces output, but the human reviews everything before it goes live. Nothing reaches a customer or system of record without human approval. This is how you build trust and catch errors early.

## Human-in-the-Loop

A design pattern where the AI does the work but a human reviews, approves, or edits the output before it takes effect. Most agents in this playbook start (and many stay) in a human-in-the-loop configuration. The human provides judgment; the AI provides speed.
