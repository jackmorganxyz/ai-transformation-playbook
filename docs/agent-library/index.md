# Agent Library

The library stays inside the docs site in v1. That keeps review, contribution,
and validation lightweight while the design standards harden.

## What every example includes

- a real workflow problem
- explicit trigger, inputs, and systems
- human approval points
- success metrics and failure modes
- security notes
- token-cost notes
- deployment mode
- a prompt skeleton

## Library structure

- nine role-specific examples across the seed verticals
- three cross-industry reference patterns for reusable workflow shapes
