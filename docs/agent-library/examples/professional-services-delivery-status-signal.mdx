## Problem Statement

Delivery leads need a current view of milestones, blockers, and follow-up. The
raw information is spread across boards, notes, and side conversations.

## Trigger

Run after a scheduled standup, project sync, or reporting checkpoint.

## Inputs

- project status fields
- open tasks and overdue milestones
- notes from the latest delivery meeting

## Systems/Tools

- project tracker
- notes workspace
- risk register

## Workflow Steps

1. Pull the latest project status data and recent notes.
2. Compare current milestone state with the prior reporting window.
3. Draft a status summary with risks, blockers, and missing updates.
4. Flag items that need owner confirmation.

## Human Approval Points

- Delivery lead confirms the risk framing.
- External-facing wording is edited before sharing with clients.

## Outputs

- internal project brief
- risk and blocker list
- follow-up prompts for owners

## Success Metrics

- reduced status-prep time
- fewer stale milestones in reports
- higher on-time owner follow-through

## Failure Modes

- hidden offline work causes inaccurate status
- ambiguous blockers are labeled too strongly
- the model misses important nuance in client promises

## Security Notes

- confine client information to approved workspaces
- preserve an audit trail for any write-back

## Token-Cost Notes

- use structured deltas instead of full project history
- cap note history to the active reporting period

## Deployment Mode

`Hybrid` fits because drafting quality matters, but review stays with the
delivery lead and nothing is sent automatically.

## Prompt Skeleton

```text
Role: Delivery operations analyst

Objective: Draft a current internal status brief for one project.

Inputs: Project status fields, milestone dates, open actions, recent meeting notes.

Constraints: Flag uncertainty, do not invent owner status, and separate confirmed risks from missing data.

Required checks: Highlight anything that needs delivery-lead review before client sharing.

Output format: Summary, risks, blockers, owner follow-ups, missing updates.
```
