## Problem Statement

Revenue teams need cleaner opportunity records and clearer next steps, but not
an autonomous system making deal judgments for them.

## Trigger

Run after a meeting summary lands or during a scheduled hygiene review.

## Inputs

- call summary
- current CRM fields
- stage requirements

## Systems/Tools

- CRM
- call summary source
- task system

## Workflow Steps

1. Compare the latest call summary against required CRM fields.
2. Draft missing field updates and owner next steps.
3. Highlight gaps, stale data, and stage quality risks.
4. Package the results for the rep or RevOps reviewer.

## Human Approval Points

- Rep approves next steps and edits language.
- Stage changes are confirmed before write-back.

## Outputs

- suggested field updates
- task list
- hygiene alerts

## Success Metrics

- faster CRM updates
- fewer stale opportunities
- better completion of next steps

## Failure Modes

- wrong interpretation of deal progress
- poor source notes create misleading tasks
- automated updates reduce trust in CRM

## Security Notes

- keep pipeline data in approved systems
- log who approved each write-back

## Token-Cost Notes

- strip filler content from call notes
- inject only the field requirements for the active stage

## Deployment Mode

`Hybrid` is appropriate because the workflow can automate preparation, but
forecast and deal decisions remain explicitly human.

## Prompt Skeleton

```text
Role: Revenue operations assistant

Objective: Suggest CRM hygiene updates and next steps after a sales interaction.

Inputs: Call summary, current opportunity record, stage rules.

Constraints: Do not change stage or forecast without human confirmation. Flag uncertainty clearly.

Required checks: Highlight missing evidence for any suggested change.

Output format: Suggested updates, next steps, hygiene warnings, confidence notes.
```
