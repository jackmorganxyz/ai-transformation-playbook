## Problem Statement

Inbound work often arrives faster than humans can classify it, but that does not
mean the right answer is immediate automation beyond routing and preparation.

## Pattern Fit

Use this when the intake item can be classified against a maintained taxonomy
and when a human still owns the downstream action. Avoid it when the taxonomy is
unstable or the first action is irreversible.

## Trigger

A new item lands in the intake queue or the item changes enough to require
re-triage.

## Inputs

- inbound payload
- routing rules
- lookup context

## Systems/Tools

- intake system
- routing taxonomy
- context lookup systems

## Workflow Steps

1. Normalize the intake payload into structured fields.
2. Classify the issue against the current taxonomy.
3. Retrieve the smallest useful context set.
4. Draft a routed summary and recommended next step.

## Human Approval Points

- Humans review low-confidence or sensitive cases.
- Financial, legal, or customer-commitment actions remain human-approved.

## Outputs

- classification label
- routed summary
- next-step recommendation

## Success Metrics

- lower manual triage time
- higher queue accuracy
- faster first-touch handling

## Failure Modes

- taxonomy drift
- overconfident routing
- sensitive exceptions hidden in generic intake text

## Security Notes

- keep context retrieval scoped to the relevant record
- retain approval history for downstream actions

## Token-Cost Notes

- favor structured fields and top excerpts over full threads
- stage long-text summarization only after classification

## Deployment Mode

`Hybrid` is usually the safest default because the pattern is best at preparing
work, not at fully closing it.

## Prompt Skeleton

```text
Role: Intake triage assistant

Objective: Classify one intake item and prepare a routed summary.

Inputs: Intake payload, taxonomy rules, relevant account or workflow context.

Constraints: Flag low confidence, avoid irreversible actions, and keep rationale visible.

Required checks: Escalate policy-sensitive or ambiguous cases.

Output format: Classification, route, summary, confidence notes, review flags.
```
