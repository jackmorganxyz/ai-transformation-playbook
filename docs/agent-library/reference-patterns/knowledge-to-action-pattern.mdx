## Problem Statement

Many workflows need a fast first draft. The draft is only useful if it stays
attached to approved knowledge and visible evidence.

## Pattern Fit

Use this when the workflow has a clear knowledge source and a human is available
to review the result. Avoid it when the underlying source is stale or the
workflow depends on uncaptured tribal knowledge.

## Trigger

A task requests a brief, reply, summary, or recommendation grounded in existing
approved material.

## Inputs

- workflow request
- approved source material
- output template

## Systems/Tools

- knowledge base
- workflow system
- draft template

## Workflow Steps

1. Retrieve the smallest relevant set of approved source passages.
2. Draft the output using only that source set.
3. Surface citations, evidence, and any unanswered questions.
4. Route the draft to a human for review.

## Human Approval Points

- Reviewer confirms that the draft matches the approved source.
- Unsupported claims are removed or sent back for more evidence.

## Outputs

- grounded draft
- evidence list
- open questions

## Success Metrics

- faster draft preparation
- higher reviewer acceptance
- fewer unsupported claims

## Failure Modes

- stale or incomplete retrieval
- grounded output that still misses material nuance
- hidden uncertainty

## Security Notes

- scope retrieval to approved repositories
- preserve provenance for each material claim

## Token-Cost Notes

- retrieve excerpts, not full documents
- keep templating separate from retrieval

## Deployment Mode

`Hybrid` is usually right because review and evidence checking matter as much as
the draft.

## Prompt Skeleton

```text
Role: Knowledge-grounded drafting assistant

Objective: Create a draft using only approved source material.

Inputs: Workflow request, approved source excerpts, output template.

Constraints: Cite evidence, flag unsupported questions, and do not invent facts outside the source set.

Required checks: Surface any missing evidence before final use.

Output format: Draft, cited evidence, open questions, reviewer flags.
```
