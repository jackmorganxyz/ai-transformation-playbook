## Problem Statement

Approvals slow down when the approver has to reconstruct the case before they
can decide. A good agent packages the case; it does not remove the approval.

## Pattern Fit

Use this when a workflow already has a defined approval step and the main issue
is missing context. Avoid it when the approval criteria are informal or when the
recommended action would be hard to reverse.

## Trigger

The workflow reaches a review gate or needs to move from one specialist to
another.

## Inputs

- task state
- supporting evidence
- approval checklist

## Systems/Tools

- workflow tracker
- notes or history store
- approval template

## Workflow Steps

1. Gather the minimum evidence set needed for a decision.
2. Summarize the current state, recommendation, and rationale.
3. Flag missing information or low-confidence areas.
4. Present the review package to the approver.

## Human Approval Points

- Approver accepts, rejects, or escalates the case.
- Downstream actions wait for the decision.

## Outputs

- review package
- recommendation
- missing-info flags

## Success Metrics

- faster approvals
- fewer repeat review cycles
- more consistent approval decisions

## Failure Modes

- omitted evidence
- recommendation anchoring
- outdated approval rules

## Security Notes

- scope the package to the approver's access level
- log the decision and rationale

## Token-Cost Notes

- summarize just the approval-relevant evidence
- reuse a structured review template

## Deployment Mode

`Hybrid` is typically correct because the pattern exists to improve human review,
not remove it.

## Prompt Skeleton

```text
Role: Approval package assistant

Objective: Prepare a concise review package for a human approver.

Inputs: Current case state, supporting evidence, approval criteria.

Constraints: Show rationale, surface missing information, and do not imply approval has already happened.

Required checks: Flag anything that makes the recommendation low confidence or hard to reverse.

Output format: Recommendation, rationale, evidence summary, missing info, decision options.
```
