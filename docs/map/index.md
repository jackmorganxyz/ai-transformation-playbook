# Map Workflows

Turn vague interest into a concrete task list. For every employee or role,
identify the specific tasks they do day to day and separate the high-value
work from the low-value execution.

## How to discover tasks

1. Pick one recurring workflow with a measurable bottleneck.
2. Identify the trigger and the final output.
3. List every step between the trigger and the output.
4. Separate prep work (low-value execution) from judgment, relationship, or
   approval work (high-value).
5. Note the systems, inputs, and exceptions at each step.

## Minimum fields per task

- workflow name
- task name
- primary owner
- trigger
- inputs
- systems touched
- output
- exception path

## Stop and re-scope if you hear

- "The AI will do the whole workflow."
- "The workflow changes every time."
- "We cannot explain who approves the result."
- "The only input is 'whatever is in the inbox.'"

## Guidance

- Map tasks at the level where an operator would recognize the work.
- Separate triggers from downstream steps.
- Keep relationship-heavy tasks distinct from clerical preparation tasks.
- Capture what system each step reads from and writes to.
- Do not score a whole department process at once.
